## 🛡️ What is AEGIS ELITE?

**AEGIS ELITE** is a fully autonomous, AI-native cybersecurity operations platform — a 111-node n8n workflow that combines 6 specialized Claude AI agents, 15+ threat intelligence sources, and 12 enterprise security layers to deliver **red team, blue team, OSINT, compliance auditing, and executive reporting** — all triggered by a single natural language command.

Unlike traditional security tools that require expert configuration, dedicated personnel, and weeks of onboarding, AEGIS ELITE understands plain English. Tell it `"we are being attacked from 185.x.x.x"` and it launches a live incident response. Tell it `"run our monthly pentest"` and a full red team engagement begins in seconds.

**In short:** AEGIS ELITE replaces a $200k+/year SOC analyst stack for approximately **$60–90/month**.

### What Makes AEGIS Truly Unique

**1. Unified Red + Blue + Compliance in One Workflow**
No other open-source project combines offensive red team automation, defensive blue team response, continuous OSINT, and multi-framework compliance auditing inside a single, self-orchestrating system. Tools like CAI and PentestGPT are offensive-only. AEGIS covers the full security lifecycle.

**3. Risk-Scored Autonomous Gating**
AEGIS does not blindly execute. Every action receives a 0–100 risk score. Low-risk actions auto-execute. CAI requires Python setup, custom agent building, and security expertise. HexStrike requires MCP server configuration. AEGIS requires importing one JSON file into n8n and replacing placeholders. A security-minded sysadmin can be fully operational in under an hour.

**5. Always-On, Self-Scheduling Intelligence**
AEGIS runs on 6 independent schedule tiers — from 15-minute breach checks to nightly red team scans — without any human involvement. It is a security operations center that never sleeps, never calls in sick, and never misses a CVE.

```bash

# 2. Click "Import Workflow"

  -H "Content-Type: application/json" \
```

## ❓ Frequently Asked Questions

<details>
<summary><strong>Do I need to be a security expert to use AEGIS ELITE?</strong></summary>

No. AEGIS ELITE accepts plain English commands. If you can describe what you want to check in a sentence, AEGIS can run it. That said, understanding the outputs (CVE scores, MITRE mappings, iptables commands) benefits from some security familiarity. The Executive Report mode is specifically designed for non-technical stakeholders.

</details>

<details>
<summary><strong>Can AEGIS ELITE scan systems I don't own?</strong></summary>

No, and this is enforced technically — not just by policy. The Scope Enforcer (Layer 4) is hardcoded to only allow actions against your verified organization domain. The Trust Score Engine and 3-Resolver DNS Verification ensure that only domains you demonstrably own can be targeted. Attempting to scan external targets will be blocked before the AI is ever invoked.

</details>

<details>
<summary><strong>What happens if the AI tries to do something dangerous?</strong></summary>

Actions with risk scores above 75/100 require explicit email approval before execution. The AI cannot override this gate — it is a code-level check, not an AI-level instruction. Additionally, all actions run through the Scope Enforcer which operates independently of AI outputs. Claude's recommendation is always subject to the upstream security layers.

</details>

<details>
<summary><strong>Is this suitable for production/enterprise environments?</strong></summary>

Yes. The enterprise JSON workflow was specifically built for production deployment. It includes: immutable audit logging for legal compliance, DLP monitoring on all outgoing data, 4-eyes approval enforcement, per-tier rate limiting, and separate HIGH vs. CRITICAL alert workflows to prevent alert fatigue. All data is processed within your n8n instance — nothing leaves to unauthorized third parties.

</details>

<details>
<summary><strong>How does AEGIS compare to hiring a penetration tester?</strong></summary>

A professional pentest engagement typically costs $5,000–$50,000+ and produces a static point-in-time report. AEGIS runs nightly automated red team scans, continuously monitors for new exposures, and delivers updated reports every Monday morning. It is not a replacement for a skilled human pentester doing creative manual testing — but for continuous baseline validation, AEGIS is unmatched at the price point.

</details>

<details>
<summary><strong>What if I get a false positive?</strong></summary>

The Defend module's AI triage includes a True Positive / False Positive classification step before any automated actions are taken. Only confirmed true positives trigger IP blocks and escalations. Additionally, all auto-block actions are reported to AbuseIPDB with a record, providing an external verification layer.

</details>

<details>
<summary><strong>Can I add my own threat intelligence sources?</strong></summary>

Yes. The n8n workflow is fully editable. You can add HTTP request nodes to any additional API, wire them into the Aggregate Intel Results node, and the AI Intel Analyst will incorporate the new data automatically. The architecture is designed to be extended.

</details>

<details>
<summary><strong>What does "No Proof, No Report" mean?</strong></summary>

The Attack module only surfaces findings it has actively validated. If a CVE is theoretically present but cannot be confirmed via API-based scanning, AEGIS will not include it in its report. This eliminates the noise of unvalidated findings that plague traditional vulnerability scanners like Nessus or OpenVAS.

</details>

<details>
<summary><strong>How do I rotate the approval token?</strong></summary>

Generate a new token with `openssl rand -hex 32`, update the `AEGIS_WEBHOOK_SECRET` environment variable in n8n, and update the `AEGIS_SECURE_2024` placeholder in the workflow. Rotate monthly as a security best practice.

</details>

<details>
<summary><strong>Is the Anthropic API key stored securely?</strong></summary>

Yes. All credentials are stored in the n8n Credentials panel using encrypted storage — never hardcoded in node parameters. The DLP Monitor also scans outgoing reports to ensure API keys are never accidentally included in delivered content.

</details>

<details>
<summary><strong>Does AEGIS work with self-hosted n8n?</strong></summary>

Yes. AEGIS ELITE works with both n8n Cloud and self-hosted n8n instances. For self-hosted deployments, ensure your instance is accessible via HTTPS with a valid SSL certificate, and that all webhook endpoints are properly secured behind your firewall.

</details>

<details>
<summary><strong>What is the per-run cost?</strong></summary>

Approximately $5 per full AUTO mode run (all modules in parallel). Individual modules cost significantly less — an INTEL-only run costs under $1. The nightly scheduled runs account for most of the monthly $30–60 Claude API spend.

</details>

## ⚖️ Legal, Ethics & Responsible Use

AEGIS ELITE is a dual-use tool. The attack capabilities it provides are functionally equivalent to those used by professional penetration testers — and by malicious actors. **Responsible deployment is non-negotiable.**

| Requirement | Detail |
|---|---|
| **Authorization** | Only run against systems you own or have explicit written permission to test |
| **Applicable Laws** | Unauthorized testing violates the CFAA (US), Computer Misuse Act (UK), and equivalent laws worldwide |
| **Approved Uses** | Own infrastructure, authorized pentests, SOC monitoring of own environment, lab research |
| **API Key Security** | Store all credentials in the n8n Credentials panel. Never hardcode keys in node parameters |
| **Transport Security** | Use HTTPS for all webhook endpoints. Never expose over plain HTTP |
| **Token Rotation** | Rotate the AEGIS_SECURE approval token monthly. Generate with: `openssl rand -hex 32` |
| **Data Privacy** | Enrichment responses may contain PII. Ensure GDPR compliance before storing |
| **Responsible Disclosure** | If third-party vulnerabilities are found, report through appropriate responsible disclosure channels |

The Scope Enforcer and Trust Score Engine provide **technical enforcement** of these policies — unauthorized scanning is blocked at the workflow level, not just by convention.
